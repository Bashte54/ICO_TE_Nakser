Patrones de Diseño en Ingeniería de Software

¿Qué son los patrones de diseño?

Los patrones de diseño son soluciones generales y reutilizables para problemas comunes en el diseño de software. Representan las mejores prácticas desarrolladas por programadores experimentados a lo largo del tiempo.

No son piezas de código listas para copiar y pegar, sino conceptos abstractos que deben adaptarse a cada situación particular. Los patrones de diseño ayudan a:

  • Acelerar el proceso de desarrollo al proporcionar paradigmas de desarrollo probados
  • Mejorar la comunicación entre desarrolladores mediante un vocabulario común
  • Crear código más mantenible y escalable
  • Evitar problemas sutiles que pueden causar complicaciones mayores

Categorías principales de patrones

Creacionales

Singleton

Asegura que una clase tenga solo una instancia y proporciona un punto de acceso global a ella.

Ejemplo de uso: Configuración global de una aplicación, conexión a base de datos.

public class Database {
    private static Database instance;
    
    private Database() {}
    
    public static Database getInstance() {
        if (instance == null) {
            instance = new Database();
        }
        return instance;
    }
}

Estructurales

Adapter

Permite que interfaces incompatibles trabajen juntas. Actúa como un puente entre dos interfaces.

Ejemplo de uso: Integración con APIs externas, compatibilidad con versiones anteriores.

interface MediaPlayer {
    void play(String audioType, String fileName);
}

interface AdvancedMediaPlayer {
    void playVlc(String fileName);
    void playMp4(String fileName);
}

class MediaAdapter implements MediaPlayer {
    AdvancedMediaPlayer advancedMusicPlayer;
    
    public MediaAdapter(String audioType) {
        if(audioType.equalsIgnoreCase("vlc")) {
            advancedMusicPlayer = new VlcPlayer();
        } else if(audioType.equalsIgnoreCase("mp4")) {
            advancedMusicPlayer = new Mp4Player();
        }
    }
    
    @Override
    public void play(String audioType, String fileName) {
        if(audioType.equalsIgnoreCase("vlc")) {
            advancedMusicPlayer.playVlc(fileName);
        } else if(audioType.equalsIgnoreCase("mp4")) {
            advancedMusicPlayer.playMp4(fileName);
        }
    }
}
Comportamiento

Observer

Define una dependencia de uno-a-muchos entre objetos, de modo que cuando un objeto cambia su estado, todos sus dependientes son notificados y actualizados automáticamente.

Ejemplo de uso: Sistemas de notificación, actualizaciones en tiempo real.

interface Observer {
    void update(String message);
}

class ConcreteObserver implements Observer {
    private String name;
    
    public ConcreteObserver(String name) {
        this.name = name;
    }
    
    @Override
    public void update(String message) {
        System.out.println(name + " recibió el mensaje: " + message);
    }
}

class Subject {
    private List observers = new ArrayList<>();
    private String state;
    
    public void addObserver(Observer observer) {
        observers.add(observer);
    }
    
    public void setState(String state) {
        this.state = state;
        notifyAllObservers();
    }
    
    private void notifyAllObservers() {
        for (Observer observer : observers) {
            observer.update(state);
        }
    }
}

Otros patrones importantes

Factory Method

Define una interfaz para crear un objeto, pero permite a las subclases decidir qué clase instanciar.

Uso típico: Cuando una clase no puede anticipar el tipo de objetos que debe crear.

Decorator

Permite añadir funcionalidad a un objeto de forma dinámica sin afectar el comportamiento de otros objetos de la misma clase.

Uso típico: Cuando necesitas extender funcionalidad de forma flexible sin usar herencia.

Strategy

Define una familia de algoritmos, encapsula cada uno de ellos y los hace intercambiables.

Uso típico: Cuando necesitas variar un algoritmo independientemente de los clientes que lo usan.

¿Cuándo usar patrones de diseño?

Los patrones de diseño son herramientas poderosas, pero deben usarse con criterio:

  • No son una solución universal: No fuerces un patrón donde no es necesario
  • Comprende el problema primero: Asegúrate de entender bien el problema antes de aplicar un patrón
  • Evita la sobreingeniería: A veces una solución simple es mejor que un patrón complejo
  • Considera el mantenimiento: Asegúrate que otros desarrolladores puedan entender tu implementación

Los patrones más utilizados en la industria incluyen Singleton, Factory, Observer, Strategy y Decorator.
